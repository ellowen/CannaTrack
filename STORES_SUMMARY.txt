ZUSTAND STORES IMPLEMENTATION SUMMARY
=====================================

CREATED/MODIFIED FILES:

1. /frontend/src/store/userStore.ts - ENHANCED
   State:
   - userId, email, name, plan (free/pro)
   - potVolumeLiters, theme (system/light/dark), notificationsEnabled, onboarded
   - totalXP, streak, bestStreak, lastActivityDate
   
   Actions:
   - setUser(userId, email, name) - batch user init
   - setName(name) | setPlan(plan) | updatePlan(plan) | setPotVolume(liters)
   - setTheme(theme) | setNotificationsEnabled(v) | setOnboarded(v)
   - updatePreferences(prefs) - batch update
   - addXP(base) - returns {xpGained, streakBonus, newStreak}
   
   Selectors:
   - getLevel() - computes current/next level from totalXP
   - getStreakBonusXP() - returns bonus for current streak
   
   Persistence: all fields via localStorage 'cannatrack-user'

2. /frontend/src/store/plantStore.ts - ENHANCED
   State:
   - plants[], selectedPlantId, loading, error, filter (active/all)
   
   Actions:
   - addPlant(plant) | updatePlant(id, changes) | removePlant(id)
   - selectPlant(id) | setLoading(v) | setError(e) | setFilter(filter)
   
   Selectors:
   - getActivePlants() - filters by status='active'
   - getPlantById(id) - find plant by ID
   - getPlantsCount() - total count
   
   Persistence: plants[], selectedPlantId, filter via localStorage 'cannatrack-plants'

3. /frontend/src/store/taskStore.ts - ENHANCED
   State:
   - tasks[], filter (all/today/overdue/completed), loading
   
   Actions:
   - addTask(task) | completeTask(id, notes) | updateTask(id, changes) | removeTask(id)
   - setTasks(plantId, tasks) | resetTasksForPlant(plantId)
   - setFilter(f) | setLoading(v)
   
   Selectors:
   - getTodayTasks() - filters by scheduledDate = today
   - getOverdueTasks() - filters by scheduled < now AND !completed
   - getCompletedCount() | getPendingCount()
   
   Persistence: tasks[] only via localStorage 'cannatrack-tasks'

4. /frontend/src/store/nutritionStore.ts - ENHANCED
   State:
   - customTables[], tables[] (computed), selectedTableId, loading
   
   Actions:
   - setTables(tables) | setSelectedTable(id)
   - addTable(table) | addCustomTable(table) - both prevent isOfficial=true
   - updateTable(id, changes) | removeTable(id) | setLoading(v)
   
   Selectors:
   - getTableById(id) - find table by ID
   - getAvailableTables(plan) - filters by plan (free gets only free tables, pro gets all)
   
   Persistence: customTables[], selectedTableId via localStorage 'cannatrack-nutrition'
   Official tables (REVEGETAR_TABLE, TOPCROP_TABLE) loaded from code, merged at rehydration

5. /frontend/src/store/syncStore.ts - NEW
   State:
   - syncQueue: SyncAction[] (actions pending online sync)
   - isSyncing, lastSyncAt (Date | null), syncError (string | null)
   
   SyncAction type:
   {
     id: string (auto-generated)
     type: 'addPlant' | 'updatePlant' | 'completeTask' | 'addXP' | 'uploadPhoto'
     payload: Record<string, unknown>
     timestamp: Date (auto-set)
   }
   
   Actions:
   - enqueueSyncAction(action) - adds to queue with auto id+timestamp
   - flushSyncQueue(onlineActions?) - clears queue, sets lastSyncAt, handles Supabase integration
   - setSyncError(error) | clearQueue() | setIsSyncing(v)
   
   Selectors:
   - getPendingActionsCount() - returns syncQueue.length
   - getLastSyncTime() - returns lastSyncAt
   
   Persistence: syncQueue (critical for offline-first) via localStorage 'cannatrack-sync'
   Implements offline-first with sync queue when connection restored

6. /frontend/src/store/index.ts - UPDATED
   Exports all store hooks:
   - usePlantStore | useTaskStore | useUserStore | useNutritionStore | useSyncStore
   Also exports SyncAction type for consumers

PERSISTENCE STRATEGY:
- All stores use Zustand persist() middleware
- Custom dateReviver deserializer handles ISO date strings
- localStorage keys: 'cannatrack-{store-name}'
- Offline-first: syncQueue persisted separately for resilience
- partialize() used where appropriate (e.g., don't persist loading/error flags)

DATE HANDLING:
- Dates stored as ISO strings in JSON (e.g., "2026-04-23T14:30:00.000Z")
- dateReviver in storage.ts auto-converts ISO strings back to Date objects
- All Date fields in state use Date type (strict TypeScript)
